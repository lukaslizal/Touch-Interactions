What's This?

Experiments with touch interaction systems in mobile games. Each of the interaction system should be very familiar to users, easy to understand and use. For that reason all of the interaction systems are implementing common touch-gestures and behaviors like Swipe, Pan, Tap or elastic scrolling with inertia. And on top of this I am exploring ways to navigate player in virtual space and help player interact with game objects.

What's That?

Project is developed in Unity game engine using TouchScript and LeanTween open-source libraries. To open this Unity project file you need Unity 5.3.2 or higher.

Orthogonal "crossroad" navigation

This system allow player move around virtual world in similar way as you know from panning through web page on your mobile device except you can move in both vertical AND horizontal ways.

Player can move along something like corridors connected to each other with "crossroad" nodes. Each of these nodes can be configured so crossroad acts like 4-way crossroad, 3-way crossroad, 2-way crossroad or dead end.

When player bumps to dead end in crossroad everything acts like elastic scrolling from iOS.

Direction can be changed only inside crossroads and user doesn't even have to lift his finger from touch screen.
